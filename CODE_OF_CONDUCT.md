# Code of Conduct

This repository is maintained for professional collaboration around public configuration distribution.

## Expected Behavior

- Be respectful and constructive.
- Keep discussions focused on the repository purpose.
- Raise risks clearly, especially around public exposure of config values.
- Give reviewers enough context to evaluate operational impact.

## Unacceptable Behavior

- Harassment, intimidation, discriminatory language, or personal attacks.
- Publishing secrets, credentials, customer-sensitive data, or internal-only operational details.
- Using public repository channels to disclose vulnerabilities or sensitive incidents.
- Repeatedly ignoring maintainer guidance for the release process.

## Enforcement

Maintainers may edit, close, or reject contributions that do not meet this standard. Serious or repeated violations may lead to restricted repository access.

Security-sensitive concerns must be reported through private Netrics security or repository-owner channels, not through public issues or pull requests.
